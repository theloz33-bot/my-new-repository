# Project Blueprint: 정치 시사 블로그\n\n## 1. 개요\n\n이 문서는 웹 컴포넌트, 최신 CSS 및 순수 JavaScript를 사용하여 현대적이고 반응이 빠르며 유지 관리가 쉬운 블로그를 만들기 위한 계획을 설명합니다.\n\n## 2. 구현된 기능\n\n*   **웹 컴포넌트 기반 UI:** 재사용 가능한 웹 컴포넌트(\`<blog-post>\`, \`<guest-book>\`)를 활용하여 UI를 모듈화했습니다.\n*   **페이지 분리:** 블로그 홈, 방명록, 공지사항, 기사 상세 보기 등 주요 기능을 별도의 페이지로 분리하여 구조를 명확히 했습니다.\n*   **동적 콘텐츠 렌더링:** 중앙 데이터 파일(`data.js`)로부터 데이터를 불러와 각 페이지의 콘텐츠를 동적으로 생성합니다.\n*   **내비게이션:** 모든 페이지에 일관된 헤더와 내비게이션 메뉴를 제공하여 사용자 이동성을 높였습니다.\n\n## 3. 파일 구조\n\n```\n.\n├── firebase.json\n├── .firebaserc\n├── blueprint.md\n├── index.html\n├── guestbook.html\n├── announcements.html\n├── article.html\n├── style.css\n├── main.js\n├── guestbook-page.js\n├── announcements-page.js\n├── article-page.js\n├── data.js\n└── components\n    ├── blog-post.js\n    └── guest-book.js\n```\n\n## 4. 현재 계획: 기사 상세 보기 및 공지사항 기능 추가\n\n블로그의 기능을 확장하여 사용자에게 더 많은 정보와 상호작용을 제공합니다.\n\n### 4.1. 목표\n\n*   **콘텐츠 식별성 강화:** 각 뉴스 기사에 고유 ID와 업로드 날짜를 추가합니다.\n*   **상세 보기 기능:** 사용자가 기사 제목을 클릭하면 해당 기사의 전체 내용을 볼 수 있는 별도의 페이지로 이동시킵니다.\n*   **공지사항 게시판 신설:** 블로그의 공지나 업데이트를 전달할 수 있는 새로운 페이지를 만듭니다.\n\n### 4.2. 상세 실행 계획\n\n1.  **데이터 구조 개편 (`data.js` 생성):\n    *   기존 `main.js`에 있던 `posts` 배열을 `data.js` 파일로 분리하여 중앙에서 데이터를 관리합니다.\n    *   `posts` 배열의 각 기사 객체에 고유한 `id`와 `date` 속성을 추가합니다.\n\n2.  **기사 상세 페이지 구현:**\n    *   **`article.html` 생성:** 단일 기사의 제목과 전체 내용을 표시하기 위한 템플릿 페이지를 만듭니다.\n    *   **`article-page.js` 생성:** URL의 쿼리 스트링(`?id=...`)에서 기사 `id`를 읽어와, `data.js`에서 해당 데이터를 찾아 페이지에 동적으로 렌더링하는 스크립트를 작성합니다.\n\n3.  **블로그 홈 페이지 수정:**\n    *   **`components/blog-post.js` 수정:** 날짜를 표시하는 UI를 추가하고, 기사 제목을 `<a>` 태그로 감싸 `article.html?id={id}`로 연결되도록 수정합니다.\n    *   **`main.js` 수정:** `data.js`로부터 기사 데이터를 `import`하여 `index.html`에 렌더링하도록 로직을 변경합니다.\n\n4.  **공지사항 페이지 생성:**\n    *   **`announcements.html` 생성:** 공지사항 게시판 역할을 할 새로운 HTML 파일을 만듭니다.\n    *   **`announcements-page.js` 생성:** 공지사항 관련 동적 로직을 처리할 스크립트 파일을 준비합니다. (초기에는 정적 콘텐츠로 구성)\n\n5.  **전체 내비게이션 업데이트:**\n    *   **`index.html`**, **`guestbook.html`**, **`article.html`**, **`announcements.html`** 파일의 헤더에 '홈', '방명록', '공지사항'으로 이동하는 내비게이션 메뉴를 일관되게 적용합니다.\n    *   **`style.css`** 에 필요시 내비게이션 관련 스타일을 보강합니다.\n\n6.  **최종 검토 및 배포:**\n    *   모든 변경 사항이 의도대로 적용되었는지, 페이지 간 이동 및 데이터 표시가 정확한지 테스트합니다.\n    *   완성된 버전을 Firebase에 배포합니다.\n
